### 4个Agent的具体设计

#### 1. Q1P0 - PainScannerAgent（痛点扫描Agent）

**职责**：持续扫描用户的即时性、持续性、隐性痛点信号

**输入源**：
- 搜索平台热词（百度/微信/小红书/抖音）
- 用户咨询记录（私信/评论/社群）
- 竞品内容分析

**输出**：
```json
{
  "pain_signals": [
    {
      "type": "instant|persistent|hidden",
      "keyword": "手机内存不足",
      "urgency_score": 95,
      "volume_trend": "rising",
      "source": "xiaohongshu_search",
      "timestamp": "2026-02-14T10:30:00Z",
      "related_questions": ["怎么办", "紧急", "马上"]
    }
  ]
}
```

---

#### 2. Q2P0 - EmotionWatcherAgent（情绪监测Agent）

**职责**：监测用户情绪状态，识别共鸣机会

**输入源**：
- 评论区情绪分析
- 社群讨论氛围
- 用户反馈聚合

**输出**：
```json
{
  "emotion_signals": [
    {
      "emotion": "anxiety|frustration|hope|anger",
      "intensity": 85,
      "trigger": "装修预算超支",
      "quotes": ["又被婆婆说了", "真的好累"],
      "source": "mom_group_chat",
      "needs_resonance": true
    }
  ]
}
```

---

#### 3. Q3P0 - TrendHunterAgent（热点追踪Agent）

**职责**：追踪平台热点，识别引流机会

**输入源**：
- 各平台热搜榜
- 话题热度变化
- 突发事件监测

**输出**：
```json
{
  "trend_signals": [
    {
      "topic": "AI写作工具",
      "platform": "douyin",
      "heat_score": 92,
      "growth_rate": "+300%",
      "lifecycle": "rising|peak|falling",
      "relevance_to_ip": 85,
      "action_window": "24h"
    }
  ]
}
```

---

#### 4. Q4P0 - SceneDiscoverAgent（场景发现Agent）

**职责**：发现用户的潜在场景，唤醒沉睡需求

**输入源**：
- 用户行为模式分析
- 季节性/周期性事件
- 跨领域场景迁移

**输出**：
```json
{
  "scene_signals": [
    {
      "scene": "年后求职准备",
      "timing": "2-3月",
      "target_user": "职场人士",
      "awaken_need": "简历优化/面试技巧",
      "trigger_events": ["年终奖发放", "春节后跳槽潮"],
      "preparation_window": "now"
    }
  ]
}
```

### Daily Sync机制

每天晚上4个Agent自动同步：

```
PainScannerAgent: "今天发现3个高urgency痛点..."
                ↓
EmotionWatcherAgent: "这些痛点的情绪底色是焦虑，Q2可以共鸣"
                ↓
TrendHunterAgent: "其中2个与当前热点契合，Q3可以借势"
                ↓
SceneDiscoverAgent: "发现这些痛点的深层场景是XX，Q4可以唤醒"
                ↓
Orchestrator: "综合决策：明天重点推送Q1内容，Q2情绪铺垫"
```

### 下一步：选择第一个实现的Agent

| 选项 | Agent | 理由 | 数据可得性 |
|------|-------|------|-----------|
| **A** | PainScannerAgent | 已有完整方法论，效果最直接 | 需要搜索API |
| **B** | EmotionWatcherAgent | 可从现有评论区开始 | 可用本地数据 |
| **C** | TrendHunterAgent | 技术最成熟，有现成API | 平台API丰富 |
| **D** | SceneDiscoverAgent | 差异化最大，竞争少 | 需要长期积累 |

#### 建议：A（PainScannerAgent）

- 你已有象限1的完整方法论
- 痛点数据最直接影响成交
- 可以为其他Agent提供输入
